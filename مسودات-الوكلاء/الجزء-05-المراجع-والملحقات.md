### 21.4 الأنماط المعمارية المتجنّبة (Anti-patterns)

| النمط | لماذا نتجنّبه |
|------|----------------|
| **God Models / God Controllers** | Model أو Controller واحد بـ500+ سطر → صعب الاختبار والصيانة. نقسّمه عبر Services, Actions, Observers. |
| **Fat Controllers** | منطق تجاري داخل Controllers → استخدم Service classes أو Action classes. |
| **N+1 Queries** | تكرار queries داخل loop → استخدم Eager Loading (`with()`) و `Model::query()->with(...)`. |
| **Mass Assignment بدون حماية** | بدون `$fillable` أو `$guarded` → ثغرة أمنية. |
| **Magic Numbers** | `if ($count > 50)` → استخدم `if ($count > Config::get('exam.max_attempts'))`. |
| **Premature Optimization** | تحسين قبل القياس → قِس أولاً بـ Laravel Pulse / Debugbar ثم حسّن. |
| **Untested Critical Paths** | لا اختبارات للمنطق المالي → 90%+ coverage إلزامي بـ Pest. |
| **Hard-coded Secrets** | في الكود → استخدم `.env` + `config()` فقط. |
| **Skipping Policies** | الاعتماد على Middleware فقط → Policies + Filament Shield طبقة دفاع ثانية. |
| **Raw SQL بدون Bindings** | عرضة لـ SQL Injection → استخدم Eloquent أو Query Builder مع bindings. |
| **استخدام `dd()` في الإنتاج** | يكشف معلومات حساسة → استخدم `Log::debug()` + إزالة قبل الـ commit. |
| **Logic داخل Blade Views** | يصعّب الاختبار → ضع Logic في ViewModels / Computed Properties في Livewire. |
