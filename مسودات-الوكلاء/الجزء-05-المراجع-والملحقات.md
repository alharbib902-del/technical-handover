### 21.4 الأنماط المعمارية المتجنّبة (Anti-patterns)

| النمط | لماذا نتجنّبه |
|------|----------------|
| **God Components** | مكوّن واحد بـ500+ سطر → صعب الاختبار والصيانة. نقسّمه. |
| **Prop Drilling** | تمرير props 5+ مستويات → استخدم Context أو Zustand. |
| **useEffect للـData Fetching** | يُسبّب race conditions → استخدم RSC أو TanStack Query. |
| **Global Mutable State** | بدون state management → استخدم Zustand مع selectors. |
| **Magic Numbers** | `if (count > 50)` → استخدم `if (count > MAX_ATTEMPTS)`. |
| **Premature Optimization** | تحسين قبل القياس → قِس أولاً ثم حسّن. |
| **Untested Critical Paths** | لا اختبارات للمنطق المالي → 90%+ coverage إلزامي. |
| **Hard-coded Secrets** | في الكود → استخدم env vars. |
| **Skipping RLS** | الاعتماد على Middleware فقط → RLS طبقة دفاع ثانية. |
| **Manual SQL Queries** | بدون validation → استخدم Zod schemas + Supabase Client. |

### 22.3 القاموس (Glossary)

#### 22.3.1 المصطلحات التقنية

| المصطلح | الشرح |
|---------|------|
| **App Router** | نظام التوجيه الجديد في Next.js 13+ مع Server Components |
| **BaaS** (Backend as a Service) | خدمة backend جاهزة (Supabase, Firebase) |
| **CDN** (Content Delivery Network) | شبكة خوادم لتوزيع الأصول قريباً من المستخدم |
| **CI/CD** | Continuous Integration / Continuous Deployment |
| **CSRF** | Cross-Site Request Forgery — هجوم |
| **DDoS** | Distributed Denial of Service — هجوم إنكار خدمة موزّع |
| **DPA** | Data Processing Agreement — اتفاقية معالجة البيانات |
| **DPIA** | Data Protection Impact Assessment — تقييم أثر حماية البيانات |
| **DPO** | Data Protection Officer — مسؤول حماية البيانات |
| **DR** | Disaster Recovery — استرداد الكوارث |
| **E2E** | End-to-End Testing — اختبار من البداية للنهاية |
| **ERD** | Entity Relationship Diagram — مخطط علاقات الكيانات |
| **GIN Index** | Generalized Inverted Index — index في Postgres للـJSONB |
| **IDOR** | Insecure Direct Object Reference — ثغرة وصول مباشر غير آمن |
| **JWT** | JSON Web Token — token مشفّر للمصادقة |
| **KMS** | Key Management Service — خدمة إدارة مفاتيح التشفير |
| **MFA / 2FA** | Multi-Factor Authentication / Two-Factor — مصادقة متعددة العوامل |
| **MTTR** | Mean Time To Recovery — متوسط وقت الإصلاح |
| **OIDC** | OpenID Connect — بروتوكول مصادقة فوق OAuth 2.0 |
| **OWASP** | Open Web Application Security Project |
| **PWA** | Progressive Web App — تطبيق ويب يعمل كتطبيق جوال |
| **PDPL** | Personal Data Protection Law — نظام حماية البيانات الشخصية السعودي |
| **RBAC** | Role-Based Access Control — تحكم وصول بناءً على الدور |
| **RLS** | Row-Level Security — أمان على مستوى الصف في Postgres |
| **RoPA** | Record of Processing Activities — سجل معالجة البيانات |
| **RPO** | Recovery Point Objective — حد الفقدان المقبول |
