🔐 الحسابات التي يستلمها المبرمج المشارك

  • دعوة GitHub للمستودع الخاص بالمشروع (Private Repo).
  • دعوة Supabase كـ Developer على المشروع.
  • إعدادات البيئة (.env) ترسل بقناة آمنة (1Password أو رسالة مشفّرة).
  • وصول لقناة التواصل (WhatsApp + قناة العمل).

📋 بروتوكول الاعتماد الصارم

⚠️ لا يبدأ العمل في أي مرحلة إلا بعد اجتياز هذا البروتوكول

  1. المبرمج المشارك يُحضّر خطة المرحلة (المهام + التصميم + الـSchema).
  2. يعرض الخطة على Codex للمراجعة والتقييم.
  3. Codex يُقيّم الخطة على 100 درجة ويعطي ملاحظات.
  4. إذا التقييم أقل من 100/100، يجب التعديل وإعادة العرض.
  5. عند الوصول لـ100/100، يُؤذَن بالبدء.
  6. أثناء التطوير: Codex يراجع الكود يومياً (Daily Review).
  7. عند الانتهاء: Codex يُقيّم النتيجة قبل تسليمها للطرف الأول.

الشهر الأول — المرحلة 1: الأساس والاختبارات

03
الشهر 1 · الأسابيع 1-4
التأسيس + منصة الاختبارات
بناء البنية التحتية الكاملة + منصة الاختبارات الإلكترونية + الشهادات المعتمدة.
1
الأسبوع 1 — التأسيس التقني
  • إنشاء مشروع Next.js 14 مع TypeScript strict mode.
  • إعداد Supabase Project + الـ Migrations الأولية.
  • تكوين TailwindCSS + shadcn/ui مع دعم RTL.
  • تثبيت خطوط Cairo + Tajawal + IBM Plex Sans Arabic.
  • إعداد ESLint + Prettier + Husky pre-commit hooks.
  • تكوين GitHub Actions CI/CD أساسي.
  • نشر أولي على Vercel للتحقق.
📦 المسلَّم نهاية الأسبوع
مشروع جاهز للتطوير + نشر يعمل على staging.
2
الأسبوع 2 — المصادقة + الفروع + RBAC
  • Supabase Auth + Email/Password + 2FA للأدوار الإدارية.
  • جداول branches + users + roles + permissions.
  • Row Level Security (RLS) على كل الجداول.
  • عزل بيانات الفروع الـ4 على مستوى DB.
  • Lockout policy (5 محاولات → قفل 15 دقيقة).
  • Middleware للحماية على المسارات.
  • Audit Log جدول + Triggers تلقائية.
📦 المسلَّم نهاية الأسبوع
تسجيل دخول لكل دور يعمل + عزل فروع مُختبَر بـRLS + Audit Log.
3
الأسبوع 3 — بنك الأسئلة (14 نوع)
  • جدول questions + question_options + question_categories.
  • تنفيذ 14 نوع سؤال (MCQ، صح/خطأ، فراغ، توصيل، سحب، Hotspot، ...).
  • واجهة إدخال الأسئلة لكل نوع.
  • استيراد Excel مع validation + تقرير أخطاء.
  • تصنيف الأسئلة (مادة / موضوع / صعوبة / تاجات).
  • البحث والفلترة المتقدمة.
  • اقتراح أسئلة AI مع موافقة المعلم.
📦 المسلَّم نهاية الأسبوع
بنك أسئلة كامل + معلم يضيف 50 سؤال + يستورد 100 من Excel.
4
الأسبوع 4 — محرك الاختبارات + الشهادات + PWA
  • محرك توليد الاختبارات (تلقائي بمواصفات / يدوي).
  • آلية كود الاختبار الذكية (Remote / Onsite).
  • التصحيح الآلي لكل أنواع الأسئلة + يدوي للمقالي.
  • 3 طبقات حماية الاختبار (Fullscreen، تبديل التابات، النسخ/اللصق).
  • نظام الدبلومات (تسلسل المواد + المعدل التراكمي).
  • توليد الشهادات تلقائياً + QR + صفحة التحقق العامة.
  • PWA للتحضير الميداني (يعمل Offline).
  • تقارير أكاديمية أساسية (Item Analysis).
📦 المسلَّم نهاية الشهر
منصة اختبارات إلكترونية كاملة + شهادات بـQR + PWA يعمل على الجوال.

الشهر 2 · الأسابيع 5-8
الطلاب والمالية والحجب ⭐
أهم شهر — إدارة الطلاب الكاملة + ربط البرنامج المحاسبي + محرك الحجب الذكي.
5
الأسبوع 5 — توسعة الطلاب والفروع
  • جدول students الكامل (شخصي + ولي أمر + توظيف اختياري).
  • State Machine للطالب (8 حالات + history table).
  • نقل الطلاب بين الفروع + سجل تاريخي.
  • دور مدير الفرع + موظف التسجيل.
  • صلاحيات تفصيلية لكل فرع.
  • شاشة بطاقة الطالب الموحّدة (7 tabs).
📦 المسلَّم نهاية الأسبوع
إدارة طلاب كاملة + 8 حالات + نقل بين الفروع.
6
الأسبوع 6 — التكامل مع البرنامج المحاسبي
  • Adapter Pattern + IAccountingService interface.
  • تنفيذ Endpoints: getStudentBalance + getPayments + getBranchRevenue.
  • جدول accounting_snapshot في Postgres للأداء.
  • Pull periodic كل 15 دقيقة (Cron).
  • Webhook receiver فوري للدفعات.
  • Reconciliation Job يومي + تنبيهات.
  • Circuit Breaker + Retry policy + Dead Letter Queue.
  • Mock Adapter للتطوير المحلي.
📦 المسلَّم نهاية الأسبوع
طبقة تكامل مكتملة + بيانات مالية حية في النظام.
7
الأسبوع 7 — Service Blocking Engine ⭐⭐
  • محرك قواعد ديناميكي (Rules Engine) قابل للتعديل من DB.
  • مصفوفة الحجب (8 حالات × 10+ خدمات).
  • حجب تلقائي بناءً على بيانات المحاسبي (تأخير، أيام، نسبة).
  • واجهة الإدارة لتعديل القواعد دون كود.
  • فك الحجب اليدوي من الإدارة + توثيق إلزامي للسبب.
  • تكامل الحجب مع الخدمات (خطابات، اختبارات، نتائج).
  • Audit Log لكل حجب وفك حجب.
📦 المسلَّم نهاية الأسبوع
محرك الحجب يعمل لحظياً + قواعد قابلة للتعديل.
8
الأسبوع 8 — لوحات المالية + التقارير
  • لوحة المالية الرئيسية (KPIs + Charts).
  • تقرير المتأخرين بأسماء وأرقام لحظياً.
  • تقارير الفروع المالية (منفصلة + مجمّعة).
  • تصدير Excel + PDF.
  • إشعارات تلقائية للطلاب قبل الاستحقاق.
  • اختبارات Integration + E2E لكل التدفق.
📦 المسلَّم نهاية الشهر
نظام مالي متكامل + محرك حجب يعمل + لوحات الإدارة.

🏛 المرحلة 4 — التكامل والذكاء (يكملها الطرف الأول)

هذه المرحلة تشمل التكاملات الحكومية والذكية، ويتولّاها الطرف الأول مباشرة بعد تسليم المبرمج المشارك.

تكامل TVTC الكامل إرسال الدرجات للوزارة لوحة الامتثال 20+ تقرير + Dashboards أرشيف المعلم تقييم المعلمين اكتشاف الطلاب المتعثرين WhatsApp Business API SMS Gateway Email Service المساعد الذكي للطلاب Nafath SSO بوابة الخريجين الدورات التطويرية

⚠️ ملاحظات صارمة

1. اعتماد Codex إلزامي

لا يُكتب سطر كود واحد قبل اعتماد Codex 100/100 لخطة المرحلة. الحضور البشري للطرف الأول لا يُغني عن اعتماد Codex.

2. تحديث الـAI Tool

إذا صدر Claude Opus 5 أو أي إصدار أحدث من Opus خلال فترة المشروع، ينتقل المبرمج لاستخدامه فوراً بعد التأكد من جاهزيته.

3. حسابات يُوفّرها الطرف الأول

GitHub + Supabase + Vercel + إعدادات البيئة (.env) تُسلَّم للمبرمج المشارك في اليوم الأول بقناة آمنة.

✅ معايير القبول لكل مرحلة

  • Codex Approval 100/100 قبل البدء + بعد الانتهاء.
  • اختبارات Unit ≥ 80٪ تغطية للمنطق التجاري.
  • اختبارات E2E للسيناريوهات الرئيسية.
  • Code Reviews يومية مع Codex.
  • RLS Policies لكل جدول جديد.
  • Audit Log للعمليات الحساسة.
  • RTL اختُبر بصرياً وعربياً.
  • Performance ضمن المعايير المتّفق عليها.
  • التوثيق محدّث (README + API docs).

🤝 إقرار المبرمج المشارك

بقراءة هذه الوثيقة، يُقرّ المبرمج المشارك بفهمه:
• أدوات العمل المعتمدة (Claude Opus + Codex).
• بروتوكول الاعتماد الصارم (Codex 100/100).
• التفاصيل التقنية للمراحل الثلاث (12 أسبوع عمل).
• أن المرحلة الرابعة خارج نطاق عمله ويُكملها الطرف الأول وحده.

هذه الوثيقة ملحق فني لعقد المبرمج المشارك — تُقرأ معه كوحدة واحدة.
