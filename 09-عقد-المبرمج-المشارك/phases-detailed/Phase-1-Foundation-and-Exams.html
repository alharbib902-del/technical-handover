لماذا هذه المرحلة قبل غيرها؟

المرحلة الأولى هي اللبنة الحجرية للمشروع كاملاً. كل مرحلة لاحقة (المالية، الحجب، الطلبات، الخطابات، الأكاديمي اليومي، TVTC، HR، التواصل الذكي) تستند إلى الأربعة أسابيع هذه. أيّ تنازل هنا يُكلّف 5—10 أضعاف لاحقاً. لهذا تُنفَّذ المرحلة باعتماد Codex 100/100 الصارم على كل قطعة قبل البدء وبعد الانتهاء.

المرحلة تجمع ثلاث طبقات متراكبة: (1) الأساس التقني والمعماري — Next.js 14 + Supabase + TailwindCSS + shadcn/ui + Auth + RBAC + RLS متعدد الفروع + Audit Log + CI/CD. (2) منصة الاختبارات الكاملة — بنك أسئلة بـ14 نوع + محرك توليد + كود الاختبار (Remote/Onsite) + التصحيح الآلي + الشهادات بـQR + PWA. (3) دورة حياة الطالب — جدول موسّع + State Machine بـ8 حالات + نقل بين الفروع + بطاقة الطالب الموحّدة + دور مدير الفرع + موظف التسجيل.

الأهداف الكبرى

  • O1. إنتاج نواة معمارية مستقرة تخدم 1,000 طالب موزّعين على 4 فروع، مع جاهزية للتوسّع إلى 10,000 طالب و8 فروع بدون إعادة كتابة.
  • O2. فرض عزل الفروع تقنياً عبر RLS صارم — مدير الفرع الأول لن يستطيع تقنياً رؤية بيانات الفرع الثاني (لا فلتر واجهة فقط).
  • O3. بناء بنك أسئلة كأصل معرفي مستقل يبقى عابراً للمعلمين والدورات والسنوات، مع 14 نوع سؤال + 3 قنوات إدخال.
  • O4. منصة اختبارات تتحمّل 100 طالب متزامن مع تصحيح آلي بدقة 100% وشهادات قابلة للتحقق العام بـQR.
  • O5. State Machine حقيقي للطالب (XState) — كل انتقال له trigger, guard, action + سجل تاريخي كامل.
  • O6. Audit Log غير قابل للحذف لكل عملية حساسة + Triggers تلقائية + امتثال PDPL مبنيٌّ في الـSchema لا مُضافٌ لاحقاً.

القرارات المعمارية المُعتمدة

  • Multi-Branch by Design: كل جدول يحوي branch_id + كل سياسة RLS تتحقق من انتماء المستخدم.
  • App Router في Next.js 14 (لا Pages Router) + Server Components أساسياً.
  • TypeScript Strict إلزامي — لا any، لا enum، استخدم as const أو type union.
  • Supabase Cloud الآن، Self-Hosted KSA مؤجَّل لمرحلة لاحقة.
  • Conventional Commits + Husky pre-commit + lint-staged + Commitlint.
  • PWA فقط — لا تطبيق جوال Native، لا متاجر تطبيقات.

📐 الفلسفة التصميمية

"بناء نواة لا تُعاد كتابتها." كل خيار تقني نتخذه هنا يُسأل أمام السؤال التالي: هل سيبقى صحيحاً عند 10,000 طالب و8 فروع؟ إن كانت الإجابة لا، لا نتخذه. الـ Multi-Tenant readiness (tenants → branches → users) قرار مُلزِم من اليوم الأول لتجنّب إعادة الكتابة عند تحويل النظام لـSaaS لاحقاً.

-- ============================================================
-- جدول audit_log (غير قابل للحذف — PDPL)
-- ============================================================
CREATE TABLE audit_log (
  id           BIGSERIAL PRIMARY KEY,
  tenant_id    UUID REFERENCES tenants(id),
  branch_id    UUID REFERENCES branches(id),
  actor_id     UUID REFERENCES users(id),
  actor_role   TEXT,
  action       TEXT NOT NULL,                   -- INSERT / UPDATE / DELETE / LOGIN / UNLOCK
  table_name   TEXT NOT NULL,
  record_id    UUID,
  old_data     JSONB,
  new_data     JSONB,
  changed_fields TEXT[],
  reason       TEXT,
  ip_address   INET,
  user_agent   TEXT,
  occurred_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_audit_actor ON audit_log(actor_id);
CREATE INDEX idx_audit_table_record ON audit_log(table_name, record_id);
CREATE INDEX idx_audit_occurred ON audit_log(occurred_at DESC);
CREATE INDEX idx_audit_branch ON audit_log(branch_id);

-- Audit Log غير قابل للحذف — PDPL إلزامي
REVOKE DELETE, UPDATE ON audit_log FROM PUBLIC, authenticated;

-- ============================================================
-- دالة Trigger عامة لتسجيل التغييرات
-- ============================================================
CREATE OR REPLACE FUNCTION audit_trigger_fn() RETURNS TRIGGER AS $$
DECLARE
  v_actor    UUID := auth.uid();
  v_role     TEXT := (SELECT role FROM users WHERE id = v_actor);
  v_branch   UUID := COALESCE(NEW.branch_id, OLD.branch_id);
  v_changed  TEXT[];
BEGIN
  IF TG_OP = 'UPDATE' THEN
    SELECT array_agg(key) INTO v_changed
    FROM jsonb_each(to_jsonb(NEW)) n
    WHERE n.value IS DISTINCT FROM (to_jsonb(OLD) -> n.key);
  END IF;

  INSERT INTO audit_log(
    actor_id, actor_role, branch_id, action,
    table_name, record_id, old_data, new_data, changed_fields
  ) VALUES (
    v_actor, v_role, v_branch, TG_OP,
    TG_TABLE_NAME, COALESCE(NEW.id, OLD.id),
    CASE WHEN TG_OP IN ('UPDATE','DELETE') THEN to_jsonb(OLD) END,
    CASE WHEN TG_OP IN ('INSERT','UPDATE') THEN to_jsonb(NEW) END,
    v_changed
  );

  RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- تطبيق الـ Trigger على الجداول الحساسة
CREATE TRIGGER trg_audit_users
  AFTER INSERT OR UPDATE OR DELETE ON users
  FOR EACH ROW EXECUTE FUNCTION audit_trigger_fn();

CREATE TRIGGER trg_audit_branches
  AFTER INSERT OR UPDATE OR DELETE ON branches
  FOR EACH ROW EXECUTE FUNCTION audit_trigger_fn();

export async function getSupabaseServer() {
  const cookieStore = cookies();

  return createServerClient(
    env.NEXT_PUBLIC_SUPABASE_URL,
    env.NEXT_PUBLIC_SUPABASE_ANON_KEY,
    {
      cookies: {
        get(name: string) {
          return cookieStore.get(name)?.value;
        },
        set(name: string, value: string, options) {
          cookieStore.set({ name, value, ...options });
        },
        remove(name: string, options) {
          cookieStore.set({ name, value: '', ...options });
        },
      },
    }
  );
}

export async function getCurrentUser() {
  const supabase = await getSupabaseServer();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return null;

  const { data: profile } = await supabase
    .from('users')
    .select('id, full_name_ar, role, branch_id, mfa_enabled')
    .eq('id', user.id)
    .single();

  return profile;
}

export async function requireRole(roles: readonly string[]) {
  const user = await getCurrentUser();
  if (!user || !roles.includes(user.role)) {
    throw new Error('FORBIDDEN');
  }
  return user;
}

🔌 Server Action — اقتراح Claude AI

src/features/questions/server/suggest-questions.ts
'use server';

import Anthropic from '@anthropic-ai/sdk';
import { z } from 'zod';
import { getSupabaseServer, requireRole } from '@/shared/lib/supabase/server';
import { env } from '@/shared/lib/env';

const InputSchema = z.object({
  category_id: z.string().uuid(),
  topic: z.string().min(3).max(200),
  difficulty: z.enum(['easy','medium','hard']),
  question_type: z.enum(['mcq_single','mcq_multi','true_false','short_answer']),
  count: z.number().int().min(1).max(10),
});

export async function suggestQuestions(input: z.infer) {
  const user = await requireRole(['instructor','super_admin','branch_manager']);
  const data = InputSchema.parse(input);

  const anthropic = new Anthropic({ apiKey: env.ANTHROPIC_API_KEY });

  const prompt = `أنت معلم خبير في إنتاج أسئلة تعليمية بالعربية.
اكتب ${data.count} سؤال من نوع "${data.question_type}" بمستوى "${data.difficulty}"
حول الموضوع: "${data.topic}".

أعد الرد بصيغة JSON صالحة فقط (بدون أي نص آخر):
{ "questions": [ { "stem": "...", "options": [...], "correct": "..." } ] }`;

  const response = await anthropic.messages.create({
    model: env.ANTHROPIC_MODEL,
    max_tokens: 4096,
    messages: [{ role: 'user', content: prompt }],
  });

  const textBlock = response.content.find(b => b.type === 'text');
  const rawText = textBlock?.type === 'text' ? textBlock.text : '';
  const parsed = JSON.parse(rawText);

  const supabase = await getSupabaseServer();
  const { data: suggestion, error } = await supabase
    .from('ai_question_suggestions')
    .insert({
      requested_by: user.id,
      category_id: data.category_id,
      prompt,
      raw_response: response,
      parsed_questions: parsed.questions,
      tokens_used: response.usage.input_tokens + response.usage.output_tokens,
    })
    .select()
    .single();

  if (error) throw new Error(`AI_SUGGESTION_SAVE_FAILED: ${error.message}`);

  return { suggestionId: suggestion.id, questions: parsed.questions };
}

type GradingResult = { score: number; max: number; details: unknown };

export function gradeQuestion(
  q: AnyQuestion & { weight: number },
  response: unknown
): GradingResult {
  switch (q.type) {
    case 'mcq_single':
      return gradeMCQSingle(q, response as { selected_option_id: string });
    case 'mcq_multi':
      return gradeMCQMulti(q, response as { selected_ids: string[] });
    case 'true_false':
      return gradeTrueFalse(q, response as { answer: boolean });
    case 'fill_blank':
      return gradeFillBlank(q, response as Record);
    case 'ordering':
      return gradeOrdering(q, response as { order: string[] });
    case 'matching':
      return gradeMatching(q, response as { pairs: Array<{l: string; r: string}> });
    case 'essay':
      return { score: 0, max: q.weight, details: { needs_manual: true } };
    // ... bocí grading functions
    default:
      return { score: 0, max: q.weight, details: { error: 'unknown_type' } };
  }
}

function gradeMCQSingle(q, r) {
  const correct = r.selected_option_id === q.answer_data.correct_option_id;
  return { score: correct ? q.weight : 0, max: q.weight, details: { correct } };
}

function gradeMCQMulti(q, r) {
  const correct = new Set(q.answer_data.correct_option_ids);
  const selected = new Set(r.selected_ids);
  const totalCorrect = correct.size;
  const correctSelected = [...selected].filter(id => correct.has(id)).length;
  const wrongSelected = [...selected].filter(id => !correct.has(id)).length;

  let score = 0;
  switch (q.answer_data.scoring_method) {
    case 'all_or_nothing':
      score = (correctSelected === totalCorrect && wrongSelected === 0) ? q.weight : 0;
      break;
    case 'partial':
      score = Math.max(0, (correctSelected / totalCorrect) * q.weight);
      break;
    case 'penalty':
      const ratio = (correctSelected - wrongSelected) / totalCorrect;
      score = Math.max(0, ratio * q.weight);
      break;
  }
  return { score, max: q.weight, details: { correctSelected, wrongSelected } };
}

import { randomBytes } from 'crypto';

const CODE_ALPHABET = 'ABCDEFGHJKMNPQRSTUVWXYZ23456789'; // بدون الأحرف المشتبهة

function generateCode(length = 6): string {
  const bytes = randomBytes(length);
  return Array.from(bytes)
    .map(b => CODE_ALPHABET[b % CODE_ALPHABET.length])
    .join('');
}

export async function generateExamCode(args: {
  exam_id: string;
  delivery: 'remote' | 'onsite';
  valid_from: string;
  valid_until: string;
  max_uses?: number;

    const code = generateCode();
    const { data, error } = await supabase
      .from('exam_codes')
      .insert({ ...args, code, is_revealed: args.delivery === 'remote' })
      .select()
      .single();

    if (!error) return data;
    if (!error.message.includes('duplicate')) throw error;
  }
  throw new Error('CODE_GENERATION_FAILED');
}

'use server';

import { z } from 'zod';
import { getSupabaseServer, requireRole } from '@/shared/lib/supabase/server';
import { revalidatePath } from 'next/cache';

const TransferSchema = z.object({
  student_id: z.string().uuid(),
  to_branch_id: z.string().uuid(),
  reason: z.string().min(20).max(500),
});

export async function requestTransfer(input: z.infer) {
  const user = await requireRole(['branch_manager','super_admin']);
  const data = TransferSchema.parse(input);
  const supabase = await getSupabaseServer();

  // 1) اقرأ الطالب + الفرع الحالي
  const { data: student, error: e1 } = await supabase
    .from('students')
    .select('id, branch_id, status')
    .eq('id', data.student_id)
    .single();

  if (e1 || !student) throw new Error('STUDENT_NOT_FOUND');
  if (student.branch_id === data.to_branch_id) throw new Error('SAME_BRANCH');

  // 2) امنع النقل إذا كان الطالب موقوفاً مالياً
  if (['suspended_financial','deprived_fees'].includes(student.status)) {
    throw new Error('TRANSFER_BLOCKED_BY_FINANCE');
  }

  // 3) أنشئ طلب نقل بحالة pending
  const { data: transfer, error: e2 } = await supabase
    .from('student_transfers')
    .insert({
      student_id: student.id,
      from_branch_id: student.branch_id,
      to_branch_id: data.to_branch_id,
      reason: data.reason,
      state: user.role === 'super_admin' ? 'approved' : 'pending',
      requested_by: user.id,
      approved_by: user.role === 'super_admin' ? user.id : null,
      approved_at: user.role === 'super_admin' ? new Date().toISOString() : null,
    })
    .select()
    .single();

  if (e2) throw new Error(`TRANSFER_CREATE_FAILED: ${e2.message}`);

  // 4) إذا super_admin اعتمد مباشرة، نفّذ النقل
  if (transfer.state === 'approved') {
    await supabase
      .from('students')
      .update({ branch_id: data.to_branch_id })
      .eq('id', student.id);

    await supabase
      .from('student_transfers')
      .update({ state: 'completed', completed_at: new Date().toISOString() })
      .eq('id', transfer.id);
  }

  revalidatePath(`/admin/students/${student.id}`);
  return transfer;
}
