كيف تسير جلسة Discovery؟

جلسة موحّدة بين إدارة المعهد وفريق التنفيذ لمراجعة الأسئلة الجوهرية، تحويل الإجابات إلى قرارات موثّقة، وتجهيز كل ما يلزم للانطلاق في الأسبوع التالي.

📋 أجندة الجلسة (2-3 ساعات)

  1. المقدمة وتقديم الفريق (15 دقيقة)

    التعريف بأعضاء الفريقين، استعراض الخطة، والاتفاق على آلية العمل اللاحق.

  2. قواعد العمل (45 دقيقة)

    مصفوفة الحجب، عتبات الحرمان، اعتماد الدرجات، شروط الاختبار الشامل، الاسترداد، الانسحاب.

  3. التكاملات التقنية (30 دقيقة)

    البرنامج المحاسبي، API الوزارة (TVTC)، النطاق، البريد، الجوال الموحّد.

  4. معلومات المؤسسة (20 دقيقة)

    الموظفون، البرامج التدريبية، الرسوم، الخصومات، اعتمادات المعهد.

  5. التشغيل والتدريب (20 دقيقة)

    أوقات الإطلاق، التدريب، الدعم بعد التسليم، إدارة IT الداخلية.

  6. الإغلاق وتحديد الخطوات التالية (15 دقيقة)

    اتفاق على دفعة البداية، تحديد نقاط الاتصال، الجدولة، نموذج الموافقة.

المحور 1: قواعد العمل

03

سياسات تشغيلية حاسمة

هذه الأسئلة تُحدد قلب النظام. أي تغيير لاحق فيها يعني إعادة بناء أجزاء كاملة.

Q1 🔴 عاجل
مصفوفة الحجب: أي خدمة تُحجب لكل حالة طالب؟
الأثر: أساس Service Blocking Engine. مثلاً: إن متأخر > 30 يوم → حجب: الخطابات / الشهادات / دخول الاختبار.
الإجابة:
Q2 🔴 عاجل
عتبات الحرمان بالغياب: ما النسبة؟ كم إنذاراً قبل الحرمان؟
الأثر: منطق الحرمان التلقائي. مثال شائع: إنذار 1 عند 5%، إنذار 2 عند 10%، حرمان عند 25%.
الإجابة:
Q3 🟠 مهم
SLA لكل نوع طلب (14 نوع): كم يوم/ساعة للمعالجة؟
الأثر: Workflow Engine يصعّد الطلبات المتأخرة تلقائياً. مثلاً: خطاب تعريف = 24 ساعة، خصم = 5 أيام.
الإجابة:
Q4 🟠 مهم
مستويات اعتماد الدرجات: من يعتمد؟ هل Override للإدارة؟
الأثر: Grade Approval Workflow. هل المشرف يعتمد ثم المدير؟ أو مستوى واحد فقط؟
الإجابة:
Q5 🟠 مهم
شروط الاختبار الشامل النهائي: متى يُعقد؟ نسبة الحضور الدنيا؟ هل يُعاد؟
الأثر: Eligibility Engine. مثال: نهاية الفصل، حضور ≥ 75%، سداد كامل.

Q9 🟢 مفيد

الأثر: منطق التحويل بين الفروع.

Q10 🟢 مفيد
سياسة الإيقاف الإداري: من يوقف؟ مدة الإيقاف؟ آلية رفعه؟
الأثر: إدارة حالات الطلاب الاستثنائية.
الإجابة:

المحور 2: التكاملات التقنية

05

جوهر الربط مع الأنظمة الخارجية

الإجابة عن هذه الأسئلة تُحدّد مسار التطوير منذ المرحلة الثانية وحتى الرابعة.

Q12 🔴 عاجل
ما اسم البرنامج المحاسبي المستخدم حالياً؟
الأثر: طبقة التكامل المحاسبي. الاسم يحدد قدرات الـAPI ونوع التكامل.
الإجابة:
Q13 🔴 عاجل
هل البرنامج المحاسبي يوفّر API REST موثّق؟ هل توجد بيئة Sandbox؟
الأثر: بدون وثائق API، لا يمكن البدء بالمرحلة 2. Sandbox ضرورية للاختبار.
الإجابة:
Q14 🔴 عاجل
حدود API المحاسبي: Rate Limits، Pagination، آلية المصادقة؟
الأثر: تصميم استراتيجية المزامنة (كل 15 دقيقة، Pagination، إلخ).
الإجابة:
Q15 🟠 مهم
هل البرنامج المحاسبي يدعم Webhooks للأحداث (مثل دفعة جديدة)؟
الأثر: Real-time sync vs Polling. Webhooks توفر استجابة فورية لعمليات الحجب.
الإجابة:
Q16 🔴 عاجل
ما اسم نظام/منصة الوزارة التي يُربط معها لرفع النتائج والحضور؟
الأثر: TVTC مباشرة؟ منصة دروب؟ نظام مخصص؟ يحدد طبيعة المرحلة 4.
الإجابة:
Q17 🔴 عاجل
هل API الوزارة موثّق؟ Sandbox متاح؟ آلية المصادقة؟
الأثر: بدون التوثيق، يُبنى Mock Adapter فقط حتى الوصول للحقيقي.

حجم المعهد ومكوّناته

Q25 🟠 مهم
عدد الموظفين الحاليين لكل دور؟ (إدارة، مالية، شؤون متدربين، تسجيل، مدربين)
الأثر: تصميم RBAC + التدريب لكل مجموعة + رخص الدخول.
الإجابة:
Q26 🟠 مهم
عدد البرامج/الدبلومات المقدمة + هياكل الرسوم لكل برنامج؟
الأثر: Seed data للنظام + التخطيط للمواد والشعب.
الإجابة:
Q27 🔵 ضروري
عدد الطلاب الفعلي لكل فرع؟ وعدد الجدد المتوقع كل فصل؟
الأثر: سعة النظام + التخطيط للأداء.
الإجابة:
Q28 🔵 ضروري
هل المعهد معتمد من TVTC؟ هل لديه اعتمادات أخرى (مثل MQF)؟

Q29 🟠 مهم

Q30 🔵 ضروري

قائمة المخرجات والخطوات القادمة

ما يخرج من الجلسة؟

بنهاية الجلسة، يجب توفّر هذه العناصر للانطلاق في الأسبوع التالي.

وثيقة الإجابات الموقّعة

هذه الوثيقة نفسها مع الإجابات الكاملة وتوقيع من المعهد.

الملفات والنماذج الرسمية للمعهد

كشف الدرجات، تقرير الحضور، الخطابات، الشهادة (Q31-Q34).

وثائق API البرنامج المحاسبي

دليل المطوّر + Sandbox credentials (Q12-Q15).

وثائق API الوزارة (TVTC)

دليل التكامل + Sandbox (Q16-Q19).

شعار المعهد بصيغ متعددة

SVG + PNG عالي الدقة + PDF (Q23).

قائمة جهات الاتصال الرسمية

4 نقاط اتصال على الأقل (تقني، مالي، إداري، أكاديمي) (Q46).

توقيع العقد الرسمي + دفعة البداية

الدفعة الأولى 5,000 ريال + توقيع وثيقة المشروع.

جدولة اجتماع الانطلاق (Kickoff)

تحديد تاريخ بداية المرحلة الأولى رسمياً.
